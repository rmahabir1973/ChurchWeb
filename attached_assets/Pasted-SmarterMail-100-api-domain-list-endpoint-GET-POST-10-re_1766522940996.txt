Based on what I found, here's the key insight: Most SmarterMail API endpoints require POST requests with a JSON body, not GET requests. That's a pro tip from a SmarterTools developer that showed up in the search - you can use the browser's network inspector to see exactly what the web interface is calling.

Let me get you the actual working code. Try this approach:

Working Domain List Request
javascript
// First authenticate
const authResponse = await fetch('https://mail.churchwebsupport.com/api/v1/auth/authenticate-user', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'your-admin@churchwebsupport.com',
    password: 'yourpassword'
  })
});

const authData = await authResponse.json();
const accessToken = authData.accessToken;

// Then get domains - this likely needs a POST with search params
const domainsResponse = await fetch('https://mail.churchwebsupport.com/api/v1/settings/sysadmin/domain-list-search', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    skip: 0,
    take: 100,
    search: null,
    sortField: 'domainName',
    sortDescending: false
  })
});

const domains = await domainsResponse.json();
console.log(domains);
Better Approach: Use Browser Dev Tools
Here's the trick a SmarterTools dev shared - the web interface uses the same API:

Log into your SmarterMail web interface at https://mail.churchwebsupport.com
Open Developer Tools (F12)
Go to the Network tab
Click to view domains in the interface
Watch the network tab - you'll see the EXACT API call being made
This will show you:

The exact endpoint URL
The HTTP method (GET/POST/etc)
The request headers
The request body (if POST)
The response format
